# Project Instructions

Read PROJECT_SPEC.md and IMPLEMENTATION_PLAN.md before making changes.

Rules:

- Implement only one phase at a time.
- Never implement future phases unless explicitly requested.
- Never modify Phase 0 processing logic unless explicitly requested.
- Reuse the processing package rather than rewriting it.
- Before implementation:
  - Explain architecture.
  - List files to be created or modified.
  - Identify risks.
  - Wait for approval.
- After implementation:
  - Explain what was built.
  - Explain how to test it.
  - List assumptions made.
  - Stop and wait for further instructions.
